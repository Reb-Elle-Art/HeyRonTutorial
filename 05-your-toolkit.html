05 — Your Toolkit

External Connections — Pick what you need. Skip what you don't.

The Toolkit Approach

Tools solve specific problems. You don't need every tool. Start with one that solves something real for you.

Philosophy: Use what serves you. Skip what doesn't. Be curious about the rest.

Each tool has its own setup page with:

  • A challenge to try first (exploration)
  • Full step-by-step instructions
  • Common problems and fixes
  • How your agent helps
  • Security considerations

How to Pick a Tool

  1. Identify a real problem — Not "what tools exist" but "what do I actually need?"
  2. Read that tool's page — Start with the "Try it first" section
  3. Work through setup with your agent — It can walk you through
  4. Test it — Make sure it actually works
  5. Come back for another tool when you need it

Don't set up everything at once. One tool mastered beats five tools half-broken.

Documenting Your Setup

Your agent resets between sessions. To help it remember:

Create an AGENT_REFERENCE.md:

# Agent Reference — My Setup

## Tools I Use

### GitHub
- Repo: github.com/yourusername/your-repo
- Token: [stored in .env, never paste here]
- Folders: backups/, uploads/, downloads/, documents/

### Other Tools
- [Add as you set them up]

## When You Need Help
1. Check tool's own setup page
2. Check this file
3. Ask: "What's my setup for [tool]?"

Tell your agent: "My toolkit is documented in AGENT_REFERENCE.md. Check there when you need setup info."

Safety Before Any Tool

  • I understand what it does
  • I trust the provider
  • I know how to turn off/revoke access
  • I won't put secrets (passwords, API keys) in plain text

Checkpoint

  • I know what the modular toolkit approach means
  • I've picked one tool that solves a real problem
  • I know where to find each tool's setup page
  • I'm ready to set it up with my agent
