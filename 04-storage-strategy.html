04 — Storage Strategy

What to Save, How Often — Don't lose your work.

The Core Principle

Don't lose your work.

Pick a backup strategy that matches how you work. Start with one. You can always add more.

Option 1: Git & Version Control

Best for: Code, project documents, configuration files, anything that changes over time.

How it works

  • Use Git locally and push to a remote repository (GitHub, GitLab, etc.)
  • Your agent can help manage commits, branches, and merges
  • You retain full history of changes

Your workflow

Start small. Push frequently. Keep commit messages clear.

Your agent handles technical Git operations if you give it permissions.

Setup commands

git init                          # Create a repo
git add .                          # Add files
git commit -m "Clear message"      # Commit regularly
git push origin main               # Push to remote

Ask your agent

"Can you help me set up a Git repository for this project?" or "Make a commit with this message: [what changed]"

Option 2: Cloud Storage & Sync

Best for: Documents, notes, media, or files that don't require version history.

How it works

  • Google Drive, Dropbox, iCloud, OneDrive — pick one
  • Automatic syncing to the cloud
  • No manual commits needed

Setup

  1. Install sync client on your machine
  2. Create organized folders (by project, by type — your choice)
  3. Make sure syncing is actually enabled

Ask your agent

"Help me organize these files into my Google Drive structure" or "What's the best way to organize my Dropbox for easy access?"

Creating Documentation for Your Agent

Your agent won't remember your backup strategy across sessions unless you document it.

Create a STORAGE_MAP.md file:

# Storage Map

## Active Projects
- Location: ~/workspace/projects/[project-name]
- Backup: Git push to origin/main weekly
- Archive: Move to ~/workspace/archive/[year] when done

## Cloud Storage
- Platform: Google Drive
- Sync folder: ~/Google Drive
- Structure: Organize by project, then by type

## Important Docs
- Keep: Local copy + Git + Cloud copy
- Review before external share: Check for API keys, PII

Point your agent to this file, and it'll know your system.

Safety Reminders: Before your agent does anything external (email, social post, public commit):

  • You verify it's going to the right place
  • You check the content (look for API keys, passwords, personal info)
  • You explicitly approve any external action

Checkpoint

  • I've chosen a backup strategy (Git, cloud, or both)
  • I understand how my backups work
  • I have documentation for my agent
  • I'm comfortable delegating file organization to my agent
  • I know how to verify external actions before they happen
