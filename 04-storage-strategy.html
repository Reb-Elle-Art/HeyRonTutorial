Your Toolkit

Tools are the hands of our agents. They reach beyond the container.

What Tools Do

Your agent lives in a container — that locked workshop you learned about in Module 03. By default, it can't touch the outside world. Tools are bridges that let it reach out: send email, post to Discord, push files to GitHub.

You control which bridges exist. Add them when you need them. Remove them when you don't.

The Toolkit Philosophy

Tools solve specific problems. You don't need every tool. Start with one that solves something real for you.

Priorities:

  • Use what serves you. Skip what doesn't.
  • Be curious about the rest.
  • Think critically about risk. Every bridge opens a door — make sure you trust what's on the other side.

Common Tools in the Community

Git & GitHub — Cloud storage, backups, version control, free website hosting
Time: 10-15 min | Skill: Beginner

  • Access from any device
  • Keep history of changes

How to Pick & Set Up a Tool

  1. Identify a real problem — Not "what tools exist" but "what do I actually need?"
    Ask your agent: "I need to [do X]. What type of tool would help?"
  2. Read the tool's documentation — Start with the setup section of a tool; better yet, have your agent read the documentation and summarize it at a level that matches your understanding
  3. Work through setup with your agent — Ask your agent to walk you through it
  4. Test it — Make sure it actually works
  5. Document the setup — So your agent remembers next session

Don't set up everything at once. One tool mastered beats five tools half-broken.

Documenting Your Setup

Your agent resets between sessions. To help it remember, work together on a reference guide.

Your container should have started with a TOOLS.md. If not, have your agent create one.

Tell your agent:

"Document this setup in TOOLS.md so you remember it next session."

Then in the future:

"Check TOOLS.md for how we configured GitHub."

Safety Before Any Tool

Before adding a bridge to the outside world, confirm:

  • [ ] I understand how it works — What does it actually do?
  • [ ] I've investigated the provider — Who makes this? Do I trust them?
  • [ ] I know how to turn it off — Can I revoke access if needed?
  • [ ] I store private information securely — API keys and passwords never go in plain text or public repos

Not sure about a tool? Ask your agent: "What risks does setting up [tool] expose me to?"

Key Takeaway

Tools extend your agent's reach. Start with one. Document it. Do your due diligence — do you trust the provider? And remember: every bridge can be crossed in both directions — think before you build.

What to Do Now

  1. Read your current TOOLS.md (or have your agent create it)
  2. Pick one tool you actually need right now
  3. Ask your agent: "What are the risks of [tool]?" (safety check)
  4. Tell your agent: "Help me set up [tool] and document it in TOOLS.md"
  5. Test it works. Stop until you need another.
