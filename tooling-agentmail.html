AgentMail — Setup

Purpose: Send and receive project emails from an agent-managed inbox. Great for notifications and small automated replies.

Skill

Beginner — API key and inbox creation required.

Estimated time

10–20 minutes

Steps

  1. Open your workspace .env and ensure AGENTMAIL_API_KEY is set.
  2. Use the provided wrapper to send a test email: 
    bash t-agentmail/send-email.sh "to@example.com" "Subject" "Body"
  3. Verify delivery and check the inbox via the AgentMail dashboard or using the check script: 
    bash t-agentmail/checkemail.sh

Verification

Successful send returns HTTP 200 and shows the message in the inbox; check agent logs for delivery confirmation.

Troubleshooting

  • Missing API key: ensure .env is sourced before running scripts.
  • Delivery failures: check spam/quarantine and logs from AgentMail provider.
