GitHub Tooling — Setup

Purpose: Repo workflows, tokens, Pages deployment, and safe push practices for agent-driven workflows.

Skill

Beginner–Intermediate

Estimated time

15–30 minutes

Setup checklist

  1. Create two GitHub tokens in Personal access tokens:
    • GITHUB_TOKEN_CLASSIC (classic PAT) — for git operations (push/pull via HTTPS).
    • GITHUB_TOKEN_API (fine‑grained) — for API calls and automation.
  2. Add tokens to your workspace .env file as GITHUB_TOKEN_CLASSIC and GITHUB_TOKEN_API.
  3. In repo operations, use the CLASSIC token embedded in the remote URL for scripted pushes: 
    git remote set-url origin https://Reb-Elle-Art:${GITHUB_TOKEN_CLASSIC}@github.com/Reb-Elle-Art/REPO.git
      git push origin main
  4. Before any public push, confirm checklist: no API keys, no personal data, and explicit approval. (User preference: agent must ask.)

Troubleshooting

  • Authentication failures: verify token scopes and that the token is the classic PAT for git operations.
  • Pages not publishing: ensure GitHub Pages settings are enabled and branch is selected in repo Settings → Pages.
