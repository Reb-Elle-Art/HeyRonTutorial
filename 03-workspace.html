Workspace_

The workspace is where your files live — and what your agent can see and modify. Keep it organized, and the agent can work effectively.

Simple Structure

You don't need complexity. A flat structure with descriptive names often works better than deep folders:

project/
  drafts/
  final/
  archive/

Naming Conventions

  • Use dashes for spaces: my-project-file.md
  • Dates as YYYY-MM-DD for sorting
  • Prefix with status: draft-, final-, archive-

What the Agent Can Access

Everything in the workspace folder. But be intentional — don't clutter it with files the agent doesn't need.

Tip: Keep a README.md in each project with a one-line description of what lives there.
