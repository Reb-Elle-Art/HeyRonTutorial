03 — Your File System

What's Where — Your agent needs to understand your environment.

Your Workspace

Your agent works from a specific location. Common setups:

Local Machine

Your laptop or desktop

  • Pro: Direct access, full control
  • Con: Dependent on your hardware staying on

Cloud VM/VPS

A remote server (AWS EC2, DigitalOcean, etc.)

  • Pro: Accessible from anywhere, always on
  • Con: Requires server management

Container

Isolated environment (Docker)

  • Pro: Reproducible, secure isolation
  • Con: Need to understand container limitations

Key question to ask your agent: "What is my current working environment, and what can you access?"

Organizing Your Files

Your agent works better when files are organized consistently.

Basic Structure

workspace/
├── projects/           # Active work
│   ├── project-1/
│   ├── project-2/
│   └── project-3/
├── reference/          # Docs, examples, resources
├── archive/            # Completed projects
└── tools/              # Scripts, configs

Naming Conventions

Keep file names clear and consistent:

  • YYYY-MM-DD_ProjectName_Description.md (for dated work)
  • ProjectName_v2_Final.md (for versioned docs)
  • Use underscores instead of spaces (easier for commands)

Where Your Agent Fits

Tell your agent your folder structure upfront:

"My workspace is organized like this: [describe it]. When I ask you to create files, put them in [appropriate folder]. When I ask you to find something, check [these locations]."

Your agent will remember and work within your system.

Special Folders

Documentation Folder

Keep tool docs, setup guides, and reference materials here:

reference/
├── tools/
├── tutorials/
└── templates/

Your agent can reference these without asking you twice.

Config Files

Store your setup info in plain text:

configs/
├── STORAGE_MAP.md       # Where files go
├── GIT_WORKFLOW.md      # Git conventions
└── AGENT_REFERENCE.md   # How to work with me

Checkpoint

  • I understand where my workspace is located
  • I know my file organization (or can describe it to my agent)
  • I have naming conventions in mind
  • I'll tell my agent about my setup
