07 Sample Project Walkthrough_

A real conversation showing how you and your agent build something together. In this case: a website to sell handmade jewelry.

The Project: Jewel & Co. Website

≫ You
I make jewelry. I want a simple website where people can see what I make and buy it. Nothing too fancy—just show my pieces and link to my Etsy shop.
࿔ Agent
Got it. Let me scope this out:
  • Static HTML site (no backend needed)
  • Portfolio of your jewelry pieces (gallery)
  • Link to Etsy for purchases
  • Mobile-responsive
We'll build it in your workspace, back it up to Github, and publish live with here.now. Start?
≫ You
Yes. Let's do it.

Step 1: Planning

࿔ Agent
I've created a folder structure:
p-jewel-co/
├── index.html
├── styles.css
├── images/
│ ├── piece-01.jpg
│ ├── piece-02.jpg
│ └── piece-03.jpg
└── README.md
What I need from you: 3-5 images of your jewelry and a short bio (2-3 sentences about you and your aesthetic).
≫ You
I'll send you the images via Telegram and here is my intro: "Handcrafted jewelry with intention. Each piece uses recycled metals and ethically sourced stones. I work with form and meaning— every design tells a story."
࿔ Agent
Perfect. I've added the images to the folder and created a draft HTML. Here's what it includes:
  • Clean header with your name/brand
  • Your bio
  • Gallery grid (3 columns on desktop, 1 on mobile)
  • Gallery descriptions
  • Link to your Etsy shop
  • Footer with contact email

Step 2: Building

≫ You
Can you show me what it looks like so far?
࿔ Agent
Here's a preview of the HTML structure:
<header>
<h1>Jewel & Co.</h1>
<p>Handcrafted jewelry with intention.</p>
</header>

<section class="gallery">
<div class="piece">
<img src="images/piece-01.jpg" alt="Ring: Copper & Agate">
<h3>Ring: Copper & Agate</h3>
<p>Minimalist form with earthy stone.</p>
</div>
...
</section>

<footer>
<a href="https://etsy.com/shop/jewel-co">Shop on Etsy</a>
<p>hello@jewelco.com</p>
</footer>
The CSS uses a dark theme (like this site) with gold accents. Responsive design built in. Want me to adjust anything?
≫ You
The dark theme is good. Can you make the gold accents more prominent on hover? And add a "About" section before the gallery.
࿔ Agent
Done. Added stronger hover effects and an "About" section with your bio. The section appears between the header and gallery. Here's the updated structure:
<section class="about">
<h2>About</h2>
<p>[Your bio]</p>
</section>
✓ About section inserted
✓ Hover effects enhanced (gold border + background)
✓ Gallery items more interactive

Step 3: Publishing

≫ You
This looks good. Let's push it live.
࿔ Agent
Before I push, confirm these steps:
  1. Create a GitHub repo called jewel-co (private)
  2. Push the project folder
  3. Publish with here.now to get a live URL
  4. Share that URL with you
All good?
≫ You
Yes. Do it.
࿔ Agent
✓ GitHub repo created: github.com/YourUsername/jewel-co
✓ Project pushed to main branch
✓ Published with here.now
Your site is live:
https://jewel-co.here.now

The site is now live. People can find it, see your work, and click through to buy on Etsy. You own the GitHub repo for future updates, and the here.now link is yours to share.

Key Takeaways

This workflow shows the full cycle: scoping → building → publishing. At each stage, your agent handles the technical work while you guide the direction. The version is backed up on GitHub, and the URL is live on here.now.

What you did:

  • Defined the goal clearly
  • Provided content (images, bio)
  • Gave feedback on design choices
  • Approved the final publish

What the agent did:

  • Scoped the technical requirements
  • Built the HTML/CSS structure
  • Managed GitHub and here.now
  • Handled all version control and deployment
