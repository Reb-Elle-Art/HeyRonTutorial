Sample Project Walkthrough

Here's what partnership actually looks like.

Example: Organizing a Mess

You: "I have 150 files spread across my desktop and three folder structures. I can't find anything, and it's slowing me down. Help me organize this."

Agent: "Okay, I need to understand your system. Can you give me examples of file types and how you'd want to find them?"

You: "Mostly docs and notes. I work on multiple projects simultaneously. Some files belong to one project, others span multiple. The biggest pain is that I can't remember where I put things."

Agent: "Got it. First, let's not try to be perfect. We'll create a simple structure with your main projects as folders, and a 'cross-project' folder for shared stuff. I'll help you move files and adjust as you see what works."

You set it up together

You: (After first setup) "This is better, but I'm losing files in the 'cross-project' folder because it's too broad. Can you add a 'reference' subfolder for things I might need later, and a 'current' subfolder for active stuff?"

Agent: "Done. Want to move some files now? Or update the system first?"

You: "Let's move them. I'll guide you."

Why This Works

  • You didn't ask your agent to solve everything
  • You broke it down
  • You gave feedback
  • You iterated
  • Each step was small enough to verify

Example: Learning a Tool

You: "I want to use [tool]. Here's the docs: [URL]. Is this something that would help with my project?"

Agent: (Reads the docs) "Yeah, it does [thing] and [thing]. For your project, it would help with [specific part]. But you'd need to [constraint]. Want me to walk you through the setup?"

You: "Yeah."

Agent: "Step 1 is [thing]. You'll need [credential/tool/setup]. Once that's done, [next step]."

You: (Completing step 1) "Done. What's next?"

Agent: "[Next step is [thing]]. Here's the command to run..."

Why This Works

  • Your agent didn't just hand you a wall of text
  • It read the docs, assessed relevance, then walked you through hands-on
  • You stayed engaged instead of drowning in documentation

The Sillage Story: How Frameworks Evolve

One community member had a problem: sessions were getting bloated. They couldn't track what they'd done across multiple conversations.

Version 1: Create a simple daily log. Each session, start by writing what's happening. At the end, summarize. Store it in a dated file.

Worked for 3 weeks. Then it broke. Everything went into the daily logs, but nothing was organized by project. Finding old decisions meant searching through weeks of logs.

Version 2: Restructure. Create project folders. Each project has its own notes. The daily log becomes a sillage-level record (what happened today), separate from project-level records (what happened in this project, ever).

Better. But it got complicated. The structure became overhead. Deciding where to put things took thought. The system was serving itself, not the work.

Version 3: Simplify. Keep what works (project folders, daily logs). Drop what doesn't (overly nested structures). Let the system be messy at first. Tidy it up only when it actually matters.

This is the version that stuck.

The lesson: You don't get the perfect system on day one. You build something. You use it. You break it. You fix it. Repeat. Each iteration teaches you what you actually need.

Your First Small Project

Pick something small. Not important, just manageable.

Week 1:

  1. Identify one problem you want your agent to help with (something that takes 30 minutes regularly)
  2. Work with your agent on that problem using the communication patterns from earlier
  3. Share context. Ask questions. Iterate once.
  4. Ship it. Use the result.

Week 2: Do another small thing. You'll know the rhythm by then.

Checkpoint

  • I see how iteration works in practice
  • I understand what "broken system" means (friction, overhead)
  • I know that v1 won't be my final system
  • I'm ready to pick my first small project
  • I understand the mindset: curious, not ChatGPT
