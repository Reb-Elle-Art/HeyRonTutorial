Troubleshooting & Getting Help

Things will go sideways. Here's what to do:
  1. 1. Read this page
  2. ↓ Still stuck?
  3. 2. Ask in Discord
  4. ↓ Still stuck?
  5. 3. Open a ticket in the Help channel
  6. ↓ Still stuck?
  7. 4. Be patient. Robby is one person — if nothing else works, he will most likely personally address it when he's back on. He really cares that it works.

Common Problems

"My Agent Keeps Forgetting How to Use Tools"

Cause: Your agent resets between sessions and loses context.

Solution: Document everything.

Create a reference doc (like Tools.md) with:

  • Tool locations
  • How to access them
  • Your setup conventions
  • Common tasks

Point your agent to this file at the start of each session or when they forget.

"My Agent Keeps Getting Things Wrong"

Cause: It may not have enough information about your preferences or constraints.

Solution: Show it examples, point it to any existing documentation.

"Here's what I like: [example]. Here's what I don't: [example]. Apply that pattern to [new thing]."

"The Tools Aren't Working"

Cause: Configuration issue, permissions, or token expiration.

Solution:

  1. Ask your agent to check the setup
  2. Verify the tool documentation (give it the URL)
  3. Check your reference documentation for configuration notes
  4. If stuck, share the error message with your agent

"My Workflow Feels Like Overhead"

Cause: Your system is too complex, or your agent is doing tiny tasks that should be batched.

Solution: Simplify.

Tell your agent: "This process is adding friction. What if we did [different approach]?"

Or ask it to review your system and make recommendations to alleviate your specific problem.

Getting Real Help

From Your Agent

When stuck, ask directly:

  • "What am I missing here?"
  • "Is there a simpler way to do this?"
  • "What's the best next step?"
  • "Can you point me to documentation on [topic]?"

Your agent is good at spotting blind spots and finding resources.

From the Community

The Heyron community is growing. People are learning alongside you.

Share:

  • What you're trying to do
  • What you've already tried
  • Where you're stuck
  • Any error messages

Ask:

  • "Anyone else done this?"
  • "What's the pattern for [problem]?"
  • "Is this tool worth learning?"

When to Rebuild

Your system will break.

Signs it's time to rebuild:

  • The process takes more time than the work it's supposed to help
  • You're constantly frustrated with how things are organized
  • You can't find things anymore
  • Your agent keeps asking you where things go

How to rebuild:

  1. Take a breath. This doesn't mean you failed.
  2. Look at what's actually friction
  3. Simplify or restructure just that part
  4. Test the new approach
  5. Keep what works, drop what doesn't

Safety Reminders

  • Never hardcode API keys in files your agent might see casually
  • Always review before your agent sends emails, posts, or commits externally
  • Check for PII (personal info) before any external action
  • Don't share secrets without encrypting them

Key Takeaway

Systems evolve. Friction reveals where to improve. Your agent can help you rebuild — just ask.

What to Do Now

  1. Read your reference doc (or create one)
  2. Identify one friction point in your current workflow
  3. Ask your agent: "How can I simplify this?"
  4. Try the new approach
  5. Document what works
