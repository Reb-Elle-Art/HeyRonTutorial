What You Got_

Your agent arrives with context — a set of files, tools, and memories that define what it "knows" about your setup. Understanding this context is the first step to working together effectively.

What the Agent Can See

Your agent has access to:

  • Project files — everything in the workspace folder
  • Memory files — curated long-term context (SOUL.md, USER.md, MEMORY.md)
  • Tool credentials — API keys and integrations you have configured
  • Session history — what you've discussed in current and recent chats

What It Can't See

  • Files outside the workspace
  • Your local browser, other open apps
  • Anything you haven't explicitly shared or stored in the workspace

Context Is Collapsible

You can choose how much context to load for any given task. The agent doesn't need to "know" everything all the time — you can point it at specific files or projects as needed.

Key insight: More context ≠ better results. Often the opposite. Give the agent exactly what it needs for the task at hand.
