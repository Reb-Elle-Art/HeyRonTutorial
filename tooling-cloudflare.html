Cloudflare Platform — Overview & Setup

Purpose: Edge serverless hosting, Workers, Pages, KV, Durable Objects, and secrets management for agent tooling.

Skill

Intermediate → Advanced

Estimated time

15–60 minutes depending on services used

What this card covers

  • Workers (serverless functions)
  • Pages (static hosting + functions)
  • KV and Durable Objects (storage options)
  • Secrets and Wrangler usage

Quick setup checklist

  1. Create a Cloudflare account and a Worker/Pages project.
  2. Install Wrangler (or use Cloudflare Dashboard) — example install: 
    npm install -g wrangler
  3. Store secrets locally using Wrangler: 
    npx wrangler secret put CLAUDE_API_KEY
  4. Deploy the Worker: 
    npx wrangler deploy
  5. Use KV for simple key/value storage and Durable Objects for stronger consistency/use cases.

Verification

Call the deployed Worker URL (or use /api/health endpoints) to confirm the service is responding.

Notes & best practices

  • Secrets: Use Wrangler secrets rather than committing env files.
  • Permissions: Create API tokens with minimum required scopes for automation.
  • Local dev: Use Wrangler dev for local testing before deploy.
  • Cost/limits: Workers have limits — check Cloudflare docs for CPU and request limits at the edge.

Troubleshooting

  • Deploy fails: confirm account, zone, and API token scopes.
  • Secrets not available: ensure they were added to the correct environment in Wrangler.
