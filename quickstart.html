QuickStart_

One-page action plan to get an agent partnership running in 15–30 minutes.

> 01 — First minute: orient

Start the conversation with a short framing line:

"Hello — I want to work together. I prefer concise, task-focused replies."

Then give one concrete task (e.g., "Fetch case 393 Mass. 564").

> 02 — First hour: try one module

Pick one area to practice: Workspace, Tools, or First Conversation. Read the short module and run one task from it (connect a single tool, fetch one file, or create one checklist).

> 03 — Tools: start small

Allow one tool at a time. Give a clear goal and constraints:

Goal: [what]
Context: [project, files]
Constraints: [what to avoid]
> 04 — Workspace basics
  • Keep project files in a single folder the agent can read.
  • Use a simple structure: project/ → drafts/, final/, archive/.
  • Never auto-publish sensitive files — require explicit confirmation before pushing public repos.
> 05 — Communication pattern

Use short commands for routine tasks and one descriptive sentence for new tasks. Ask for progress checks only when needed. Correct the agent concisely — it will learn.

> 06 — Sample micro-project

Goal: Create a 1-page summary of a document.

  1. Agent: fetch the document
  2. Agent: extract 5 key points
  3. Agent: write a 3-sentence summary
  4. You: review and mark the process repeatable
> 07 — Troubleshooting
  1. Ask the agent to show what it did (logs / commands / files)
  2. Re-run with stricter constraints
  3. If still failing, stop any publish actions and ask for a short failure report
> 08 — Next steps

Add one new tool or module per week. Keep a short "how we work" note in the project root.
