Heyron QuickStart

A single‑page action plan to get an agent partnership running in 15–30 minutes.

  1. First minute — orient
    Start the conversation with a short framing line: "Hello — I want to work together. I prefer concise, task‑focused replies." Then give one concrete task (e.g., "Fetch case 393 Mass. 564").
  2. First hour — try one module
    Pick one area to practice: Workspace, Tools, or First Conversation. Read the short module and run one task from it (connect a single tool, fetch one file, or create one checklist).
  3. Tools — start small
    Allow one tool at a time. Give a clear goal and constraints using this mini‑template:
    Goal: [what]
    Context: [project, files]
    Constraints: [what to avoid]
  4. Workspace basics — quick checklist
    • Keep project files in a single folder the agent can read.
    • Use a simple structure: project/ → drafts/, final/, archive/.
    • Never auto‑publish sensitive files — require explicit confirmation before pushing public repos.
  5. Communication pattern
    Use short commands for routine tasks and one descriptive sentence for new tasks. Ask for progress checks only when needed. Correct the agent concisely — it will learn.
  6. Sample micro‑project (15–30 minutes)
    Goal: Create a 1‑page summary of a document.
    1. Agent: fetch the document.
    2. Agent: extract 5 key points.
    3. Agent: write a 3‑sentence summary and save to /project/summaries/.
    4. You: review and mark the process repeatable.
  7. Troubleshooting (quick)
    1. Ask the agent to show what it did (logs / commands / files).
    2. Re‑run with stricter constraints (smaller scope, fewer tools).
    3. If still failing, stop any publish actions and ask for a short failure report.
  8. Next steps
    Add one new tool or module per week, keep a short "how we work" note in the project root, and set a 1‑line rule for auto‑actions (e.g., Never push to public repos without explicit approval).
